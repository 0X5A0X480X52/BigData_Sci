# 数据清洗与获取模块设计

## 架构概览

```bash
ETL Framework (数据获取与清洗/)
│
├── config.py                 # 全局配置（PyAlex、MySQL、缓存、日志）
├── main.py                   # ETL 主入口，提供 OpenAlexETL 类和 CLI
│
├── sources/                  # 数据源统一管理
│   ├── base.py              # BaseSource 抽象基类
│   └── openalex.py          # OpenAlexSource（基于 pyalex 库）
│   └── ...                  # 未来可扩展: ArxivSource, CrossRefSource
│
├── cleaners/                 # 实体级清洗器（单一职责）
│   ├── base.py              # BaseCleaner 抽象基类
│   ├── work.py              # WorkCleaner - 论文字段提取
│   ├── author.py            # AuthorCleaner - 作者字段提取
│   ├── institution.py       # InstitutionCleaner - 机构字段提取
│   ├── concept.py           # ConceptCleaner - 概念/关键词提取
│   └── venue.py             # VenueCleaner - 期刊/会议提取
│
├── pipelines/                # 清洗流水线（组合 Cleaner，适配数据源）
│   ├── base.py              # BasePipeline, CleanedResult, BatchCleanedResult
│   └── openalex.py          # OpenAlexPipeline（适配 pyalex JSON 结构）
│   └── ...                  # 未来可扩展: ArxivPipeline
│
├── cache/                    # 缓存/备份管理
│   ├── base.py              # CacheManager 抽象基类
│   ├── file_cache.py        # FileCacheManager（本地 JSON 文件）
│   └── mongo_cache.py       # MongoCacheManager（预留接口）
│
├── db/                       # 数据库操作
│   ├── connection.py        # MySQLConnection（连接管理、事务）
│   └── inserter.py          # MySQLInserter（按外键顺序批量插入）
│
├── models/                   # 数据实体定义
│   └── entities.py          # dataclass 实体（WorkEntity, AuthorEntity 等）
│
└── docs/                     # 文档
    └── 数据清洗与获取模块设计.md
```

## 核心组件说明

### 1. 数据源层 (sources/)

- **BaseSource**: 定义统一接口 `get_work()`, `search_works()`, `get_author()`, `get_institution()`
- **OpenAlexSource**: 封装 pyalex 库，支持分页、缓存集成、速率控制

### 2. 清洗器层 (cleaners/)

- **单一职责**: 每个 Cleaner 只负责提取一种实体类型
- **BaseCleaner**: 提供通用工具方法 `safe_get()`, `clean_string()`, `extract_openalex_id()`
- 不处理实体间关联关系，关联关系由 Pipeline 处理

### 3. 流水线层 (pipelines/)

- **Pipeline**: 适配特定数据源的 JSON 结构
- **OpenAlexPipeline**: 处理 pyalex 返回的嵌套结构
  - 解析 `authorships[]` → 提取作者、机构、建立关联
  - 解析 `concepts[]` → 提取概念、权重
  - 解析 `primary_location` → 提取期刊信息
- **CleanedResult**: 单条记录的清洗结果
- **BatchCleanedResult**: 批量处理结果，自动去重

### 4. 缓存层 (cache/)

- **统一接口**: `save()`, `load()`, `exists()`, `list_keys()`
- **FileCacheManager**: 按 `{entity_type}/{id}.json` 存储
- **MongoCacheManager**: 预留接口，后续实现

### 5. 数据库层 (db/)

- **MySQLConnection**: 连接管理、事务、外键检查控制
- **MySQLInserter**: 按外键依赖顺序插入
  - 顺序: countries → work_types → institutions → authors → venues → concepts → works → 关联表
  - 使用 `INSERT IGNORE` 处理重复

## 使用示例

```python
from main import OpenAlexETL, ETLConfig

# 配置
config = ETLConfig()
config.pyalex.email = "your_email@example.com"
config.mysql.password = "your_password"

# 执行 ETL
etl = OpenAlexETL(config)
stats = etl.run(
    query="machine learning",
    filters={"publication_year": 2023},
    max_results=100
)
print(stats)
```

## 命令行使用

```bash
python -m 数据获取与清洗.main -q "deep learning" -y 2023 -m 50 \
    --email your@email.com --db-pass your_password
```

## 快速使用说明

以下是一些常见的快速命令：

1. 仅获取并清洗（跳过数据库写入）：

```bash
python -m 数据获取与清洗.main -q "machine learning" -m 10 --email your@email.com --no-db
```

1. 递归抓取引用（获取直接引用的论文，带进度条）：

```bash
python -m 数据获取与清洗.main -q "machine learning" -m 5 -d 1 --max-citations 50 --email your@email.com --no-db
```

1. 将结果写入 MySQL：

```bash
python -m 数据获取与清洗.main -q "deep learning" -m 50 --email your@email.com --db-host localhost --db-user root --db-pass yourpass --db-name Scientific_Info_db
```

1. 从缓存读取并入库：

```bash
python -m 数据获取与清洗.main --from-cache --no-db
```

提示：使用 `--citation-depth`（`-d`）控制递归层数，使用 `--max-citations` 限制每篇论文最多抓取的引用数以避免抓取规模爆炸。
